Introduction

Welcome to the Permissioned Test Net for the Logos Network.

We thank you, first and foremost, for your interest in and engagement with our project, and we hope that you are impressed with the progress we continue to make after only a year since our inception.

When we set out to create the world’s premier hyperscalable, real-time and low-cost payment platform, the goal seemed like a hefty one. Yet, after rigorous mathematical calculations and the assembling of top-shelf engineering talent, we were well on our way.

Ultimately, tackling transactions costs of credit card companies and the cumbersome structure of banks, Logos was created as a way to streamline payments for consumers and merchants.

By trying out the Logos test net, we hope you see that our mission to build the payment platform of the future, today, is now becoming a reality.

This manual will instruct on how to interact with the test net and highlight all the aspects of our current development. Please note that the manual gets progressively more technical.

We look forward to your feedback, and thank you again for being a part of the journey.

Node Explorer

The Node Explorer will be your gateway into viewing what is occurring within the Logos Network. You can access the Node Explorer at https://www.pla.bs.

There is currently a temporary password – locoforlogos – on the Node Explorer.

Explore

Explore is the home page of the Logos Node Explorer. Here, you can see the most recent blocks and transactions.

The search bar will allow you to enter the hash of any block or the address of any Logos account and press the return key to find information about it.

Accounts

The accounts page allows you to dive into all the transactions an account has made or received, as well as see the balance of the account.

Transactions

The transaction page allows you to inspect the contents of a transaction, including who it is from, who it is sent to, the transaction amount, and the previous receive or send hash.

Blockchains

The chains page of the Node Explorer will allow you to see the consensus mechanisms Logos is using.

The first tab is Batch Blocks, which are delegate specific, so you can use the select input to choose an individual delegate you would like to see.

Continue reading below for more information about each type of block.

Batch Blocks

Batch Blocks are created by delegates and are a way of scaling the network. Batch Blocks, in their current implementation, can contain up to 1,500 transactions which the delegate will propose for consensus between all other delegates every 2-5 seconds.

Each delegate will have its own Batch Blocks chain.

Micro Epochs

Micro Epochs contain summaries of the Batch Blocks that occurred in a 10 minute portion of the epoch, similar to the basic epoch blocks. These Micro Epochs would periodically be issued by the delegate assigned to produce the Full Epoch block and validated by the other delegates via standard consensus. Micro Epochs allow the Full Epochs to contain a reference to the Micro Epoch chain rather than the transaction summary data.

In addition, nodes are able to periodically ensure they are fully synched to the latest 10-minute Micro Epoch rather than the last 12-hour epoch.

Epochs

Epochs contain a reference to the Micro Epoch chain for the last 12 hours. Epochs are used for the node to quickly sync to the active network.

In addition, Epoch transition will be the time in which new delegates are elected and old delegates - retired.

Faucet

The faucet is your gateway to participating in the Logos Network test net.

Simply enter your address into the input or use the QR scanner. The faucet will grant you 0.0001% of the faucet balance up to a maximum of 1,000 Logos.

The faucet is only available for the test net and the Logos coins you obtain during the test net will disappear.

Workbench

My Account

The ‘My Account’ section of the Workbench allows you to provide a private key and query information about that account. This private key is sent over SSL to the specified node URL.

Please note that all actions provided in this section can be done using other APIs that do not require your private key.

Create a Send

Use this method to send funds from one account to another:

  • Logos Amount
  • Account ID / Address of the recipient
  • Optional Force Send to Delegate
    • Forcing the send to be processed by the specified delegate will incur a five-second confirmation delay. This is an incentive to load balance the transactions as to not overload a single delegate.

Troubleshooting

There are a couple of known bugs that we are working on resolving with our front-end facing systems.

iOS

  1. Currently on the iOS app there is an error that can occur when creating a send.

    • One of these errors may occur when sending the full amount of Logos in your wallet as you are unable to cover the full cost of the transaction including the transaction fee. The solution for now is to simply send less than your full balance.
    • The second error will occur each time we reset the logos network which will cause the iOS Wallet’s state to be out of sync with the real network causing forks which are rejected by our consensus mechanism. Follow the images below to show how to reset your wallet. First on the home page select the settings gear. Second choose clear wallet data.


  2. There is currently also an issue with not recieving real-time data to the iOS Wallet make sure your MQTT sever url is correct. First on the home page select the settings gear. Second choose the server URLs options. Enter into the MQTT server url wss://pla.bs:8443
